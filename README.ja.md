# ReqLock

[English](README.md) · **日本語** · [简体中文](README.zh_CN.md) · [Español](README.es_ES.md) · [Deutsch](README.de_DE.md) · [Français](README.fr_FR.md) · [فارسی](README.fa_IR.md)

*別表記として **RequestLock** または **Request Lock**。ペルシャ語 / فارسی: **رک لاک (ریکوئست لاک)**。*

> WordPress コア、テーマ、プラグインからの**外部（インターネット）呼び出しを無効化する**マスターキルスイッチ型 WordPress プラグインです。外部インターネットが遮断・制限されてもサイトを動かし続けます。

接続が制限された状況での回復力のために作られています。スイッチを 1 つ切り替えるだけで、サイトはインターネットへのアクセスをやめ、ローカル資産のみから配信します。さらにおまけとして、ブロックされた呼び出しはタイムアウトを待たずに即座に失敗するため、**wp-admin が応答のない外部リクエストでハングしなくなります**。

> **障害時だけのものではありません** — **遅いネットワークリクエスト**への一般的な対策としても機能します。インターネットが通常どおり使えるときでも、WordPress の**フロントエンドおよび管理画面（バックエンド）**の性能を引き下げる、遅い・不安定なサードパーティ呼び出しを遮断します。ブロックされた呼び出しは即座に失敗するため、タイムアウトを待ってページ読み込みを止めることがありません。

## 遮断する対象

**サーバー側（PHP / WP HTTP API）**
- 外部ホストへの送信 `wp_remote_*` リクエスト：WordPress.org の更新・バージョン確認、アナリティクス、OpenAI / Gemini API、リモートフォントなど。
- 外部呼び出しは（タイムアウトせずに）*即座に*失敗 → オフライン時の管理画面が高速になります。

**ブラウザー側（描画された HTML）**
- 外部 `<script src="…">`
- 外部 `<link rel="stylesheet">`（例：Google Fonts）
- リソースヒント：`preconnect` / `dns-prefetch` / `preload` / `prefetch`
- 外部 `<iframe>` → クリーンなローカルのプレースホルダーに置換
- インラインのアナリティクススニペット：Google Analytics / Tag Manager、Microsoft Clarity、Ahrefs、Meta Pixel、Hotjar、Yandex Metrica
- *(任意)* 外部 `<img>` → 透明なプレースホルダー

## 主な動作

- **自分のドメインとそのすべてのサブドメインは常に許可されます**（例：`my.yoursite.com`）。
- **許可リスト**：到達可能なまま残したい他のホストを追加できます。
- **検出ホストパネル**：プラグインが見たすべての外部ホストが記録されるため、許可リストをすばやく構築できます。
- **カテゴリー別スイッチ**：各遮断レイヤーを個別にオン／オフできます。
- **オフ時は不活性**：マスタースイッチがオフのとき、プラグインは何もしません。インストールしたまま永続的に置いておき、必要なときだけオンにしても安全です。
- **管理バーのインジケーター**が ReqLock の有効状態を表示します。

## インストール

1. `reqlock` フォルダーを `wp-content/plugins/` にコピーします。
2. プラグイン画面から **ReqLock** を有効化します。
3. **設定 → ReqLock** を開きます。
4. 外部インターネットが利用できないときに、**マスタースイッチ**をオンにします。

## 設定の概要

| グループ | オプション | 既定値 |
|---|---|---|
| マスター | ReqLock | **オフ** |
| サーバー側 | 送信 WP HTTP API を遮断 | オン |
| ブラウザー側 | 外部スクリプト | オン |
| ブラウザー側 | 外部スタイルシート | オン |
| ブラウザー側 | リソースヒント（preconnect/dns-prefetch/…） | オン |
| ブラウザー側 | 外部 iframe | オン |
| ブラウザー側 | インラインのアナリティクススニペット | オン |
| ブラウザー側 | 外部画像 | オフ |
| 範囲 | wp-admin もサニタイズ | オフ |
| 範囲 | 検出ホストを記録 | オン |
| — | ホストの許可リスト | 空 |

## 要件

- WordPress 5.0 以上
- PHP 7.2 以上（7.4 / 8.1 / 8.2 でテスト済み）

## 仕組み

- **HTTP API**：`pre_http_request` にフックし、（許可リストを尊重しつつ）あらゆる外部ホストに対して `WP_Error` を返すことで、リクエストがサーバーを離れる前に短絡させます。
- **エンキューされた資産**：ソースが外部にある登録済みスクリプト／スタイルをデキュー／登録解除します。
- **描画された HTML**：ページ出力をバッファリングし、保守的でしっかりアンカーされたパターンを用いて外部リソースタグと既知のインラインアナリティクススニペットを取り除きます。取り除かれた各要素は追跡可能な `<!-- ReqLock blocked … -->` コメントを残します（リクエストは一切行われません）。

外部の `rel="canonical"` / `alternate` / アイコン `<link>` は意図的にそのまま残し（SEO に安全）、相対／内部 URL には決して手を触れません。

> 注意：WordPress を完全に経由しないスタンドアロンの PHP エントリーポイント（例：生の `curl` を使う独自スクリプト）は WordPress プラグインでは捕捉**できない**ため、外部呼び出しは各自で管理する必要があります。
