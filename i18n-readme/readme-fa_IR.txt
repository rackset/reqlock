فایروالِ درخواست‌های خروجی برای وردپرس: مسدودسازی درخواست‌های خارجی و کندِ شخص‌ثالث برای تاب‌آوری، کارایی و حریم خصوصی — تنها با یک کلید.

== Description ==

«**ReqLock**» — که **RequestLock** یا **Request Lock** نیز نوشته می‌شود (فارسی: رک لاک / ریکوئست لاک) —
یک فایروالِ درخواست‌های خروجی (egress) برای وردپرس است. این افزونه همهٔ فراخوانی‌هایی را که سایت شما
*به بیرون* و به‌سوی اینترنت انجام می‌دهد، هم در سمت سرور و هم در سمت مرورگر کنترل می‌کند. یک کلید اصلی،
سه کاربرد:

* **تاب‌آوری** — هنگام **قطع یا محدودشدن** اینترنت بین‌الملل (اختلال، خاموشی) سایت را پابرجا نگه می‌دارد. صفحه‌ها از منابع محلی سرویس داده می‌شوند و پیشخوان وردپرس دیگر روی درخواست‌های بی‌پاسخ معطل نمی‌ماند.
* **کارایی** — فراخوانی‌های کند یا بی‌پاسخِ شخص‌ثالث **بی‌درنگ ناموفق می‌شوند** و به‌جای انتظار برای تایم‌اوت، بارگذاری صفحه‌های بخش کاربری و پیشخوان (بک‌اند) را معطل نمی‌کنند.
* **حریم خصوصی** — تحلیل‌گرها، ردیاب‌ها، فونت‌های خارجی و فراخوانی‌های phone-home را حذف می‌کند.

= چه چیزهایی را مسدود می‌کند =

**سمت سرور (PHP / WP HTTP API)**

* درخواست‌های خروجیِ `wp_remote_*` به میزبان‌های خارجی: بررسی به‌روزرسانی/نسخهٔ WordPress.org، تحلیل‌گرها، APIهای هوش مصنوعی (OpenAI، Gemini)، فونت‌های راه‌دور و غیره. به‌جای تایم‌اوت، فوری ناموفق می‌شوند.

**سمت مرورگر (HTMLِ رندرشده)**

* `<script src>` خارجی و `<link rel="stylesheet">` خارجی (مثلاً Google Fonts)
* Resource hintها: `preconnect` / `dns-prefetch` / `preload` / `prefetch`
* `<iframe>` خارجی (با یک جای‌نگه‌دار محلی جایگزین می‌شود)
* اسنیپت‌های تحلیلیِ درون‌خطی: Google Analytics / Tag Manager، Microsoft Clarity، Ahrefs، Meta Pixel، Hotjar، Yandex
* اختیاری: `<img>` خارجی (جای‌نگه‌دار شفاف)

= رفتار کلیدی =

* دامنهٔ خود سایت و همهٔ زیردامنه‌هایش **همیشه مجاز** هستند.
* **فهرست مجاز** برای هر میزبان دیگری که باید در دسترس بماند.
* **پنل میزبان‌های شناسایی‌شده** که هر میزبان خارجیِ دیده‌شده را ثبت می‌کند تا به‌سرعت فهرست مجاز خود را بسازید.
* **کلیدهای جداگانه برای هر دسته** — هر لایهٔ مسدودسازی را مستقل روشن/خاموش کنید.
* **خاموش = بی‌اثر** — وقتی کلید اصلی خاموش است افزونه هیچ کاری نمی‌کند، بنابراین می‌توانید نصب نگهش دارید و فقط هنگام نیاز روشنش کنید.
* **سازگار با کش تمام‌صفحه** — فیلتر خروجی به‌عنوان بیرونی‌ترین بافر اجرا می‌شود و بنابراین بازدیدهای کش‌شده را هم پوشش می‌دهد.
* **کنترل تعارض در wp-config** — مقدار ثابتِ `WP_HTTP_BLOCK_EXTERNAL` را تشخیص می‌دهد و امکان می‌دهد آن را با یک کلیک غیرفعال (کامنت) یا دوباره فعال (بازگردانی) کنید تا ReqLock تنها کلید کنترل مسدودسازی خارجی باشد.

== Installation ==

۱. پوشهٔ `reqlock` را در `/wp-content/plugins/` بارگذاری کنید، یا فایل ZIP را از مسیر **افزونه‌ها ← افزودن ← بارگذاری افزونه** نصب کنید.
۲. از صفحهٔ **افزونه‌ها** افزونهٔ **ReqLock** را فعال کنید.
۳. به **تنظیمات ← ReqLock** بروید.
۴. هر زمان خواستید درخواست‌های خارجی مسدود شوند (هنگام قطعی، یا برای قطع فراخوانی‌های کند/ردیابی)، **کلید اصلی را روشن کنید**. پیش‌فرض **خاموش** است، پس فعال‌کردن به‌تنهایی چیزی را تغییر نمی‌دهد.

== Frequently Asked Questions ==

= آیا فعال‌کردن آن سایتم را خراب می‌کند؟ =
خیر. وقتی کلید اصلی خاموش است افزونه کاملاً بی‌اثر است. حتی وقتی روشن باشد، دامنهٔ خود سایت و زیردامنه‌هایش همیشه مجاز هستند.

= چه زمانی باید کلید اصلی را روشن کنم؟ =
هر زمان بخواهید سایت را از سرویس‌های خارجی جدا کنید: هنگام قطعی/محدودیت اینترنت، برای جلوگیری از کندشدن بارگذاری به‌دلیل فراخوانی‌های کندِ شخص‌ثالث، یا برای حذف ردیاب‌ها به‌خاطر حریم خصوصی.

= آیا با افزونه‌های کش کار می‌کند؟ =
بله. فیلتر خروجی به‌عنوان بیرونی‌ترین بافر خروجی اجرا می‌شود و بنابراین بازدیدهای کش‌شده را هم فیلتر می‌کند (با افزونه‌های کش تمام‌صفحه آزمایش شده است).

= آیا بر پیشخوان (wp-admin) اثر می‌گذارد؟ =
مسدودسازیِ درخواست‌ها در سمت سرور همه‌جا اعمال می‌شود (که هنگام قطع شبکه، پیشخوان را سریع‌تر می‌کند). پاک‌سازیِ HTML در سمت مرورگر به‌صورت پیش‌فرض فقط در بخش کاربری اجرا می‌شود؛ به‌صورت اختیاری می‌توانید آن را برای پیشخوان هم فعال کنید.

= چطور یک سرویس خارجی را فعال نگه دارم و بقیه را مسدود کنم؟ =
میزبان آن را به فهرست مجاز اضافه کنید. پنل میزبان‌های شناسایی‌شده هر چیزی را که ReqLock می‌بیند فهرست می‌کند تا بتوانید از همان‌جا کپی کنید.

= تکلیف اسکریپت‌های سفارشیِ PHP که وردپرس را دور می‌زنند چیست؟ =
اسکریپت‌های مستقلی که خارج از وردپرس از `curl`/`file_get_contents` خام استفاده می‌کنند، توسط افزونه قابل‌رهگیری نیستند و باید فراخوانی‌های خارجیِ خود را خودشان کنترل کنند.
