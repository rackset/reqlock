فایروالِ درخواست‌های خروجی برای وردپرس: مسدودسازی درخواست‌های خارجی و کندِ شخص‌ثالث برای تاب‌آوری، کارایی، حریم خصوصی و توسعهٔ آفلاین — تنها با یک کلید.

== Description ==

«**ReqLock**» — که **RequestLock** یا **Request Lock** نیز نوشته می‌شود (فارسی: رک لاک / ریکوئست لاک) —
یک فایروالِ درخواست‌های خروجی (egress) برای وردپرس است. این افزونه هر فراخوانی‌ای را که سایت شما
*به بیرون* و به‌سوی اینترنت انجام می‌دهد، در هر دو سمتِ درخواست کنترل می‌کند: **سمت سرور**
(PHP / WP HTTP API) و **سمت مرورگر** (همان HTMLی که صفحه‌های شما رندر می‌کنند). یک کلید اصلی،
سایت شما را به‌طور کامل بر ترافیک خروجیِ خودش مسلط می‌کند.

سایت‌های امروزیِ وردپرس «پرسروصدا» هستند: بررسی‌های به‌روزرسانی، پینگ‌های مجوز، تحلیل‌گرها،
تگ‌منیجرها، فونت‌های خارجی، ابزارک‌های جاسازی‌شده، APIهای هوش مصنوعی و انواع فراخوانی‌های «phone-home»
همگی به سرورهایی دست می‌زنند که شما کنترلی بر آن‌ها ندارید. وقتی این سرورها کند، مسدود یا از کار افتاده
باشند، صفحه‌ها و پیشخوان شما تاوانش را می‌دهند — و هر یک از آن‌ها مجرایی برای نشت دادهٔ بازدیدکنندگان
شماست. ReqLock به شما امکان می‌دهد این ترافیک را هر وقت بخواهید، بی‌درنگ و به‌صورت برگشت‌پذیر قطع کنید،
بدون ویرایش فایل‌های قالب و بدون جست‌وجوی افزونه‌ها.

**یک کلید، چهار کاربرد:**

* **تاب‌آوری** — هنگام **قطع یا محدودشدنِ** اینترنت بیرونی (اختلال، خاموشی منطقه‌ای، خرابیِ بالادست) سایت را پابرجا نگه می‌دارد. صفحه‌ها از منابع محلی سرویس داده می‌شوند و پیشخوان وردپرس دیگر روی درخواست‌های بی‌پاسخ معطل نمی‌ماند.
* **کارایی** — فراخوانی‌های کند یا بی‌پاسخِ شخص‌ثالث **بی‌درنگ ناموفق می‌شوند** و به‌جای معطل‌کردنِ بارگذاری صفحه‌های بخش کاربری و پیشخوان (بک‌اند) با تایم‌اوت‌های طولانی، فوراً کنار می‌روند.
* **حریم خصوصی** — تحلیل‌گرها، ردیاب‌ها، فونت‌های خارجی و فراخوانی‌های phone-home را حذف می‌کند تا هیچ چیزی دربارهٔ بازدیدکنندگانتان از سرور خارج نشود.
* **توسعه** — هر نصبی را به یک محیطِ خودبسنده و **آفلاین‌پذیر** تبدیل می‌کند: بدون فراخوانی خارجی، بدون ردیابی از یک نسخهٔ استیجینگ، و بدون انتظار برای APIهای راه‌دور هنگام کار.

= موارد کاربرد =

* **تاب‌آوری در برابر قطعی/خاموشی.** وقتی اتصال بالادست محدود یا مسدود می‌شود، یک سایت معمولیِ وردپرس روی هر فراخوانی خارجی گیر می‌کند. ReqLock را روشن کنید تا سایت — از جمله پیشخوان — همچنان از منابع محلی سرویس داده شود.
* **سرعت‌بخشیدن به یک سایت کند.** تنها یک میزبانِ کندِ تحلیل یا فونت می‌تواند چند ثانیه به هر بارگذاری اضافه کند. ReqLock این فراخوانی‌ها را به‌جای مسدودکردنِ رندر، به‌سرعت ناموفق می‌کند.
* **استقرارهای حریم‌خصوصی‌محور / بدون ردیابی.** سایتی را اجرا کنید که بتوان اثبات کرد هیچ درخواست شخص‌ثالثی نمی‌فرستد — مناسب برای پروژه‌های حریم‌خصوصی‌محور، ابزارهای داخلی و پیکربندی‌های حساس به انطباق.
* **توسعهٔ محلی و استیجینگ.** نسخهٔ تولید را روی لپ‌تاپ یا سرور استیجینگ کلون کنید و ReqLock مانع «phone-home» آن می‌شود: هیچ تحلیلی از یک نسخهٔ آزمایشی شلیک نمی‌شود، هیچ پینگ مجوزی نیست، و هیچ بررسیِ به‌روزرسانیِ WordPress.org پیشخوان (`wp-admin`) را هنگام ساخت کند نمی‌کند. سایت حتی با کابلِ شبکهٔ جداشده هم همان رفتار را دارد — ایده‌آل برای کدنویسی آفلاین، دموها و ماشین‌های ایزوله.
* **ممیزیِ این‌که سایت با کجا حرف می‌زند.** پنل میزبان‌های شناسایی‌شده هر میزبان خارجی‌ای را که سایت به آن می‌رسد ثبت می‌کند، تا دقیقاً ببینید قالب‌ها و افزونه‌هایتان با چه کسی تماس می‌گیرند — و سپس تصمیم بگیرید چه چیزی مجاز و چه چیزی قطع شود.

= چه چیزهایی را مسدود می‌کند =

**سمت سرور (PHP / WP HTTP API)**

* درخواست‌های خروجیِ `wp_remote_*` به میزبان‌های خارجی: بررسی به‌روزرسانی/نسخهٔ WordPress.org، تحلیل‌گرها، APIهای هوش مصنوعی (OpenAI، Gemini)، فونت‌های راه‌دور، پینگ‌های مجوز/phone-home و غیره. به‌جای تایم‌اوت، فوری ناموفق می‌شوند.

**سمت مرورگر (HTMLِ رندرشده)**

* `<script src>` خارجی و `<link rel="stylesheet">` خارجی (مثلاً Google Fonts)
* Resource hintها: `preconnect` / `dns-prefetch` / `preload` / `prefetch`
* `<iframe>` خارجی (با یک جای‌نگه‌دار محلی جایگزین می‌شود)
* اسنیپت‌های تحلیلیِ درون‌خطی: Google Analytics / Tag Manager، Microsoft Clarity، Ahrefs، Meta Pixel، Hotjar، Yandex، TikTok، Pinterest، LinkedIn، X، Snap، Segment، Plausible
* اختیاری: `<img>` خارجی (جای‌نگه‌دار شفاف)

= رفتار کلیدی =

* دامنهٔ خود سایت و همهٔ زیردامنه‌هایش **همیشه مجاز** هستند.
* **فهرست مجاز** برای هر میزبان دیگری که باید در دسترس بماند.
* **پنل میزبان‌های شناسایی‌شده** که هر میزبان خارجیِ دیده‌شده را ثبت می‌کند تا به‌سرعت فهرست مجاز خود را بسازید.
* **کلیدهای جداگانه برای هر دسته** — هر لایهٔ مسدودسازی را مستقل روشن/خاموش کنید.
* **کنترل تعارض در wp-config** — ثابتِ هاردکدشدهٔ `WP_HTTP_BLOCK_EXTERNAL` را تشخیص می‌دهد و امکان می‌دهد آن را با یک کلیک غیرفعال (کامنت) یا دوباره فعال (بازگردانی) کنید تا ReqLock تنها کلید کنترل مسدودسازی خارجی باشد. تغییرها برگشت‌پذیر، دارای بررسی یکپارچگی و اتمیک‌اند.
* **خاموش = بی‌اثر** — وقتی کلید اصلی خاموش است افزونه هیچ کاری نمی‌کند، بنابراین می‌توانید نصب نگهش دارید و فقط هنگام نیاز روشنش کنید.
* **سازگار با کش تمام‌صفحه** — فیلتر خروجی به‌عنوان بیرونی‌ترین بافر اجرا می‌شود و بنابراین بازدیدهای کش‌شده را هم پوشش می‌دهد.

== Installation ==

۱. پوشهٔ `reqlock` را در `/wp-content/plugins/` بارگذاری کنید، یا فایل ZIP را از مسیر **افزونه‌ها ← افزودن ← بارگذاری افزونه** نصب کنید.
۲. از صفحهٔ **افزونه‌ها** افزونهٔ **ReqLock** را فعال کنید.
۳. به **تنظیمات ← ReqLock** بروید.
۴. هر زمان خواستید درخواست‌های خارجی مسدود شوند (هنگام قطعی، برای قطع فراخوانی‌های کند/ردیابی، یا برای آفلاین‌کردنِ یک نسخهٔ استیجینگ/محلی)، **کلید اصلی را روشن کنید**. پیش‌فرض **خاموش** است، پس فعال‌کردن به‌تنهایی چیزی را تغییر نمی‌دهد.

== Frequently Asked Questions ==

= آیا فعال‌کردن آن سایتم را خراب می‌کند؟ =
خیر. وقتی کلید اصلی خاموش است افزونه کاملاً بی‌اثر است. حتی وقتی روشن باشد، دامنهٔ خود سایت و زیردامنه‌هایش همیشه مجاز هستند.

= چه زمانی باید کلید اصلی را روشن کنم؟ =
هر زمان بخواهید سایت را از سرویس‌های خارجی جدا کنید: هنگام قطعی/محدودیت اینترنت، برای جلوگیری از کندشدن بارگذاری به‌دلیل فراخوانی‌های کندِ شخص‌ثالث، برای حذف ردیاب‌ها به‌خاطر حریم خصوصی، یا برای آفلاین‌کردنِ کاملِ یک نسخهٔ استیجینگ/محلی هنگام توسعه.

= آیا می‌توانم برای توسعهٔ آفلاین از آن استفاده کنم؟ =
بله — این یک کاربرد محوری است. کلید اصلی را روشن کنید تا آن نصب دیگر به WordPress.org، تحلیل‌گرها، سرورهای مجوز، فونت‌ها و دیگر میزبان‌های راه‌دور دست نزند. سپس سایت محلی یا استیجینگ شما حتی با شبکهٔ جداشده هم به همان شکل بارگذاری و رفتار می‌کند و هرگز از یک نسخهٔ غیرتولیدی فراخوانیِ ردیابی یا phone-home شلیک نمی‌کند.

= آیا با افزونه‌های کش کار می‌کند؟ =
بله. فیلتر خروجی به‌عنوان بیرونی‌ترین بافر خروجی اجرا می‌شود و بنابراین بازدیدهای کش‌شده را هم فیلتر می‌کند (با افزونه‌های کش تمام‌صفحه آزمایش شده است).

= آیا بر پیشخوان (wp-admin) اثر می‌گذارد؟ =
مسدودسازیِ درخواست‌ها در سمت سرور همه‌جا اعمال می‌شود (که هنگام قطع شبکه، پیشخوان را سریع‌تر می‌کند). پاک‌سازیِ HTML در سمت مرورگر به‌صورت پیش‌فرض فقط در بخش کاربری اجرا می‌شود؛ به‌صورت اختیاری می‌توانید آن را برای پیشخوان هم فعال کنید.

= چطور یک سرویس خارجی را فعال نگه دارم و بقیه را مسدود کنم؟ =
میزبان آن را به فهرست مجاز اضافه کنید. پنل میزبان‌های شناسایی‌شده هر چیزی را که ReqLock می‌بیند فهرست می‌کند تا بتوانید از همان‌جا کپی کنید.

= تکلیف اسکریپت‌های سفارشیِ PHP که وردپرس را دور می‌زنند چیست؟ =
اسکریپت‌های مستقلی که خارج از وردپرس از `curl`/`file_get_contents` خام استفاده می‌کنند، توسط افزونه قابل‌رهگیری نیستند و باید فراخوانی‌های خارجیِ خود را خودشان کنترل کنند.
