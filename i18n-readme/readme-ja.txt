WordPress 向け送信ファイアウォール。外部および遅いサードパーティのリクエストを 1 つのスイッチでブロックし、耐障害性・パフォーマンス・プライバシーを実現。

== Description ==

**ReqLock**（別名 **RequestLock** / **Request Lock**。ペルシア語: رک لاک / ریکوئست لاک）は、
WordPress のための送信（アウトバウンド／egress）ファイアウォールです。サイトがインターネットへ
*送信* するすべての呼び出しを、サーバー側・ブラウザー側の両方で制御します。1 つのマスタースイッチで
3 つの用途に対応します。

* **耐障害性** — 外部インターネットが**遮断・制限**されたとき（障害、シャットダウン）でもサイトを維持します。ページはローカルのアセットから配信され、wp-admin が応答しないリクエストで固まることもありません。
* **パフォーマンス** — 遅い、または応答しないサードパーティ呼び出しが、タイムアウトを待たずに**即座に失敗**し、フロントエンドと管理画面（バックエンド）のページ読み込みを妨げません。
* **プライバシー** — 解析ツール、トラッカー、外部フォント、ホームへの通信（phone-home）を除去します。

= ブロックする対象 =

**サーバー側（PHP / WP HTTP API）**

* 外部ホストへの送信 `wp_remote_*` リクエスト（WordPress.org の更新・バージョン確認、解析、AI API（OpenAI、Gemini）、リモートフォントなど）。タイムアウトを待たずに即座に失敗します。

**ブラウザー側（レンダリングされた HTML）**

* 外部の `<script src>` と外部の `<link rel="stylesheet">`（例: Google Fonts）
* リソースヒント: `preconnect` / `dns-prefetch` / `preload` / `prefetch`
* 外部 `<iframe>`（ローカルのプレースホルダーに置換）
* インライン解析スニペット: Google Analytics / Tag Manager、Microsoft Clarity、Ahrefs、Meta Pixel、Hotjar、Yandex
* （任意）外部 `<img>`（透明なプレースホルダー）

= 主な動作 =

* 自分のドメインとそのすべてのサブドメインは**常に許可**されます。
* 到達可能なまま維持したいその他のホスト用の**許可リスト**。
* 検出されたすべての外部ホストを記録する**検出ホストパネル**。許可リストをすばやく作成できます。
* **カテゴリーごとの切り替え** — 各ブロック層を個別にオン／オフできます。
* **オフ時は不活性** — マスタースイッチがオフのときは何もしないため、インストールしたままにして必要なときだけオンにできます。
* **フルページキャッシュ上でも動作** — 出力フィルターは最も外側のバッファーとして動作するため、キャッシュされたページの表示もカバーします。
* **wp-config の競合制御** — ハードコードされた `WP_HTTP_BLOCK_EXTERNAL` を検出し、ワンクリックで無効化（コメントアウト）・再有効化（復元）できます。これにより ReqLock が外部ブロックの唯一のスイッチになります。

== Installation ==

1. `reqlock` フォルダーを `/wp-content/plugins/` にアップロードするか、**プラグイン → 新規追加 → プラグインのアップロード**から ZIP をインストールします。
2. **プラグイン**画面で **ReqLock** を有効化します。
3. **設定 → ReqLock** を開きます。
4. 外部リクエストをブロックしたいとき（障害時や、遅い／トラッキング呼び出しを止めたいとき）に**マスタースイッチをオン**にします。既定はオフなので、有効化しただけでは何も変わりません。

== Frequently Asked Questions ==

= 有効化するとサイトが壊れますか？ =
いいえ。マスタースイッチがオフのときプラグインは完全に不活性です。オンのときでも、自分のドメインとそのサブドメインは常に許可されます。

= マスタースイッチはいつオンにすべきですか？ =
サイトを外部サービスから切り離したいときならいつでも：インターネットの障害・制限時、遅いサードパーティ呼び出しが読み込みを遅くするのを止めたいとき、プライバシーのためにトラッカーを除去したいとき。

= キャッシュ系プラグインと併用できますか？ =
はい。出力フィルターは最も外側の出力バッファーとして動作するため、キャッシュされたページの表示もフィルタリングします（フルページキャッシュ系プラグインで検証済み）。

= wp-admin に影響しますか？ =
サーバー側のリクエストブロックはどこでも適用されます（ネットワーク停止時に管理画面が高速になります）。ブラウザー側の HTML サニタイズは既定でフロントエンドのみですが、wp-admin にも適用できます。

= 一部の外部サービスだけを許可したいときは？ =
そのホストを許可リストに追加します。検出ホストパネルに ReqLock が検出したすべてのホストが表示されるので、そこからコピーできます。

= WordPress を経由しないカスタム PHP スクリプトは？ =
WordPress の外で生の `curl`／`file_get_contents` を使うスタンドアロンのスクリプトはプラグインから捕捉できないため、それぞれ自身で外部呼び出しを制御する必要があります。
