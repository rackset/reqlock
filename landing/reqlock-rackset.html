ReqLock

An outbound firewall for WordPress. Block external & slow third-party requests — for resilience, performance, privacy, and offline development — with one switch.

Modern WordPress sites are noisy — update checks, license pings, analytics, external fonts, AI APIs and other phone-home calls all reach servers you don't control. When those are slow or blocked, your pages and dashboard pay for it. ReqLock puts your site in full control of its own outbound traffic, on both the server (PHP / WP HTTP API) and the browser (rendered HTML) — instantly and reversibly, no theme edits.

One switch, four jobs

Outage resilience · performance · privacy · development

🛡️

Resilience

Keep the site working when the internet is cut or restricted. Pages serve from local assets and wp-admin stops hanging on dead requests.

⚡

Performance

Slow or dead third-party calls fail instantly instead of stalling front-end and back-end page loads on long timeouts.

🔒

Privacy

Strip analytics, trackers, external fonts and phone-home requests so nothing about your visitors leaves the server.

💻

Development

Turn any install into a self-contained, offline-capable environment — no external calls or tracking from a staging copy.

What it blocks

Both sides of every request

Server-side (PHP / WP HTTP API)

  • Outbound wp_remote_* calls to external hosts
  • WordPress.org update / version checks
  • Analytics, AI APIs (OpenAI, Gemini), remote fonts
  • License / phone-home pings — fail fast, no timeouts

Browser-side (rendered HTML)

  • External <script> & <link> stylesheets (e.g. Google Fonts)
  • Resource hints: preconnect / dns-prefetch / preload / prefetch
  • External iframes (local placeholder)
  • Inline analytics: GA/GTM, Clarity, Meta Pixel, Hotjar, and more

Key features

 

  • Your own domain & subdomains are always allowed
  • Allow-list for any host that must stay reachable
  • Detected-hosts panel to audit what your site contacts
  • Per-category toggles — each layer independent
  • wp-config conflict control: disarm / re-arm WP_HTTP_BLOCK_EXTERNAL
  • Inert when OFF — safe to keep installed, flip on when needed
  • Works over full-page caches (outermost output buffer)
  • 7 languages bundled (en, ja, zh_CN, es, de, fr, fa)

Free & open source

GPL-2.0. Install from the WordPress plugin directory or grab the source on GitHub.
